## ADDED Requirements

### Requirement: Swagger UI is served in development
The application SHALL serve the Swagger interactive API reference at `/swagger` when running in the `Development` environment, using the OpenAPI document generated by `AddSwaggerGen()`.

#### Scenario: Swagger UI accessible in development
- **WHEN** the application runs in the `Development` environment
- **AND** a browser navigates to `/swagger`
- **THEN** the Swagger UI page SHALL load and display all documented endpoints

#### Scenario: Swagger UI not served in production
- **WHEN** the application runs in the `Production` environment
- **AND** a request is made to `/swagger`
- **THEN** the response SHALL return 404

### Requirement: Swagger is registered via UseSwaggerUI
The application SHALL call `app.UseSwagger()` and `app.UseSwaggerUI()` inside the `if (app.Environment.IsDevelopment())` block, and SHALL NOT use `MapScalarApiReference` or any Scalar middleware.

#### Scenario: Swagger middleware registered correctly
- **WHEN** the application starts in development
- **THEN** the route `/swagger` SHALL be mapped and serve the Swagger HTML page

### Requirement: Auth endpoints are visible in Swagger UI
The Swagger UI SHALL display an "Authorize" button for the JWT Bearer security scheme so that developers can authenticate and test protected endpoints directly from the UI.

#### Scenario: Bearer token input available in Swagger UI
- **WHEN** a developer opens the Swagger UI at `/swagger`
- **THEN** an "Authorize" button SHALL be present
- **AND** the developer SHALL be able to enter a Bearer token
- **AND** subsequent requests from the UI SHALL include the `Authorization: Bearer <token>` header

### Requirement: JWT security scheme declared in SwaggerGen
The application SHALL register a `SecurityDefinition` of type `Http` with scheme `bearer` and `bearerFormat` `JWT` inside `AddSwaggerGen()`, and SHALL add a global `SecurityRequirement` referencing that definition.

#### Scenario: Security definition present in generated OpenAPI document
- **WHEN** the application generates the Swagger document
- **THEN** the document SHALL contain a `securitySchemes` entry named `Bearer`
- **AND** all operations SHALL reference it via a global security requirement
