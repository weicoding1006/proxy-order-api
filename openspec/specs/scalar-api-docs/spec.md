### Requirement: Scalar UI is served in development
The application SHALL serve the Scalar interactive API reference at `/scalar/v1` when running in the `Development` environment, using the OpenAPI document generated by `AddOpenApi()`.

#### Scenario: Scalar UI accessible in development
- **WHEN** the application runs in the `Development` environment
- **AND** a browser navigates to `/scalar/v1`
- **THEN** the Scalar API reference page SHALL load and display all documented endpoints

#### Scenario: Scalar UI not served in production
- **WHEN** the application runs in the `Production` environment
- **AND** a request is made to `/scalar/v1`
- **THEN** the response SHALL return 404

### Requirement: Scalar is registered via MapScalarApiReference
The application SHALL call `app.MapScalarApiReference()` inside the `if (app.Environment.IsDevelopment())` block to register the Scalar endpoint, and SHALL NOT use any third-party Swagger/Swashbuckle middleware.

#### Scenario: Scalar endpoint registered correctly
- **WHEN** the application starts in development
- **THEN** the route `/scalar/v1` SHALL be mapped and serve the Scalar HTML page

### Requirement: Auth endpoints are visible in Scalar
The Scalar UI SHALL display the security scheme (Bearer JWT) so that developers can authenticate and test protected endpoints directly from the UI.

#### Scenario: Bearer token input available in Scalar
- **WHEN** a developer opens the Scalar UI
- **THEN** a security section SHALL allow entering a Bearer token
- **AND** subsequent requests from the UI SHALL include the `Authorization: Bearer <token>` header
