### Requirement: Swagger UI is served in development
The application SHALL serve the Swagger interactive API reference when running in the `Development` environment, using the OpenAPI document generated by `AddOpenApi()`.

#### Scenario: Swagger UI accessible in development
- **WHEN** the application runs in the `Development` environment
- **AND** a browser navigates to the Swagger UI endpoint
- **THEN** the Swagger UI page SHALL load and display all documented endpoints

#### Scenario: Swagger UI not served in production
- **WHEN** the application runs in the `Production` environment
- **AND** a request is made to the Swagger UI endpoint
- **THEN** the response SHALL return 404

### Requirement: Swagger UI is registered via UseSwaggerUI
The application SHALL call `app.UseSwaggerUI()` inside the `if (app.Environment.IsDevelopment())` block to register the Swagger UI endpoint, and SHALL NOT use Scalar or other third-party API reference middleware.

#### Scenario: Swagger UI endpoint registered correctly
- **WHEN** the application starts in development
- **THEN** the Swagger UI route SHALL be mapped and serve the Swagger HTML page

### Requirement: Auth is visible in Swagger UI
The Swagger UI SHALL display the security scheme (Bearer JWT) so that developers can authenticate and test protected endpoints directly from the UI.

#### Scenario: Bearer token input available in Swagger UI
- **WHEN** a developer opens the Swagger UI
- **THEN** an Authorize button or security section SHALL allow entering a Bearer token
- **AND** subsequent requests from the UI SHALL include the `Authorization: Bearer <token>` header

### Requirement: JWT security scheme is declared in SwaggerGen
The application SHALL register a JWT Bearer security scheme and security requirement in `AddSwaggerGen()` so that the Authorize button appears and all protected endpoints show the lock icon.

#### Scenario: Security scheme registered in SwaggerGen
- **WHEN** the application configures Swagger services
- **THEN** `AddSwaggerGen()` SHALL include a `SecurityDefinition` for `Bearer` using `SecuritySchemeType.Http` with scheme `bearer` and `bearerFormat` `JWT`
- **AND** a global `SecurityRequirement` SHALL reference the `Bearer` scheme so all endpoints require it by default
